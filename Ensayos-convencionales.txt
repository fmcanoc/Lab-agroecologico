# Pestaña "Análisis Externo" — Carga manual

Guía para agregar el registro de **ensayos convencionales** (análisis de laboratorios externos como INTA: NPK, materia orgánica, pH, CE, sales, etc.) a la plataforma **Laboratorio Abierto Agroecológico**, respetando los patrones que ya tenés en `app.py` y `templates/index.html`.

El diseño usa **un esquema flexible**: la metadata del informe en columnas fijas y las **determinaciones guardadas como JSON** en un campo de texto. Así no importa qué laboratorio sea ni qué determinaciones traiga — cualquier estructura entra sin tocar la base.

> **Núcleo (PASOS 1 a 5):** carga manual completa y funcional, 100% gratis.
> **Extra opcional (PASOS 6 a 8):** contraste contra valores propios, export CSV y asistente.

---

## PASO 1 — Crear la tabla (en `app.py`)

Dentro de la función `crear_tablas()`, sumá este bloque junto a los otros `CREATE TABLE` (por ejemplo, justo después del bloque de `respiracion_suelo`). Seguí el mismo patrón de `try/except` con `rollback`:

```python
            try:
                cur.execute('''CREATE TABLE IF NOT EXISTS analisis_externo (
                    id SERIAL PRIMARY KEY,
                    muestra_id INTEGER REFERENCES muestras(id) ON DELETE CASCADE,
                    laboratorio TEXT,
                    nro_protocolo TEXT,
                    fecha_recepcion DATE,
                    fecha_informe DATE,
                    archivo_url TEXT,
                    observaciones TEXT,
                    determinaciones TEXT,
                    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP)''')
                conexion.commit()
            except Exception:
                conexion.rollback()
```

**Por qué `determinaciones TEXT`:** ahí se guarda un JSON con la lista de determinaciones, igual que ya hacés con `curva_tiempo`/`curva_co2` en `respiracion_suelo`. Cada determinación es un objeto `{parametro, valor, unidad, metodo, nivel}`. Una muestra = un análisis externo (se reemplaza con DELETE+INSERT, como en `ph_conductividad`).

## PASO 5 — El JavaScript de las determinaciones dinámicas (en `templates/index.html`)

Cerca de los otros `<script>` del final del archivo, agregá:

```html
    <script>
    (function () {
        const cont   = document.getElementById('det-rows');
        const hidden = document.getElementById('det-json');
        const form   = document.getElementById('form-analisis-externo');
        const addBtn = document.getElementById('det-add');
        if (!cont || !form) return;

        function addRow(d) {
            d = d || {};
            const div = document.createElement('div');
            div.className = 'row g-2 mb-2 det-row align-items-center';
            div.innerHTML =
                '<div class="col-md-3"><input type="text" list="lista-parametros" class="form-control form-control-sm det-parametro bg-light" placeholder="Parámetro" value="' + (d.parametro || '') + '"></div>' +
                '<div class="col-md-2"><input type="text" class="form-control form-control-sm det-valor bg-light" placeholder="Valor" value="' + (d.valor || '') + '"></div>' +
                '<div class="col-md-2"><input type="text" class="form-control form-control-sm det-unidad bg-light" placeholder="mg/kg, dS/m..." value="' + (d.unidad || '') + '"></div>' +
                '<div class="col-md-3"><input type="text" class="form-control form-control-sm det-metodo bg-light" placeholder="Método" value="' + (d.metodo || '') + '"></div>' +
                '<div class="col-md-1"><input type="text" class="form-control form-control-sm det-nivel bg-light" placeholder="Nivel" value="' + (d.nivel || '') + '"></div>' +
                '<div class="col-md-1 text-end"><button type="button" class="btn btn-sm btn-outline-danger det-del"><i class="bi bi-x-lg"></i></button></div>';
            cont.appendChild(div);
            div.querySelector('.det-del').addEventListener('click', function () { div.remove(); });
        }

        // Serializa las filas al JSON oculto (antes de enviar y en cada cambio, por si el form se guarda offline)
        function serializar() {
            const rows = [];
            cont.querySelectorAll('.det-row').forEach(function (r) {
                const parametro = r.querySelector('.det-parametro').value.trim();
                const valor     = r.querySelector('.det-valor').value.trim();
                if (!parametro && !valor) return;
                rows.push({
                    parametro: parametro,
                    valor: valor,
                    unidad: r.querySelector('.det-unidad').value.trim(),
                    metodo: r.querySelector('.det-metodo').value.trim(),
                    nivel:  r.querySelector('.det-nivel').value.trim()
                });
            });
            hidden.value = JSON.stringify(rows);
        }

        addBtn.addEventListener('click', function () { addRow(); });
        cont.addEventListener('input', serializar);
        form.addEventListener('submit', serializar);

        addRow(); // arranca con una fila vacía
    })();
    </script>
```

**Con esto ya funciona la carga manual.** Lo de abajo es opcional.

## PASO 7 — (Opcional) Limpieza al borrar y ruta de eliminación

### 7a. Ruta para eliminar un análisis externo (en `app.py`)

Agregala junto a las otras rutas `/eliminar_...`:

```python
@app.route('/eliminar_analisis_externo/<int:muestra_id>', methods=['POST'])
def eliminar_analisis_externo(muestra_id):
    if 'usuario_id' not in session: return redirect(url_for('login'))
    conexion = obtener_conexion()
    cur = conexion.cursor()
    try:
        cur.execute('''DELETE FROM analisis_externo WHERE muestra_id = %s
                       AND muestra_id IN (SELECT id FROM muestras WHERE usuario_id = %s)''',
                    (muestra_id, session['usuario_id']))
        conexion.commit()
        flash('Análisis externo eliminado.', 'success')
    finally:
        cur.close()
        conexion.close()
    return redirect(url_for('inicio') + '#analisis-externo')
```

### 7b. Limpieza al borrar una muestra (en `app.py`)

El `ON DELETE CASCADE` ya lo borra solo, pero para mantener la consistencia con tu ruta `eliminar_muestra`, podés sumar la línea junto a los otros DELETE:

```python
        cur.execute('DELETE FROM analisis_externo WHERE muestra_id = %s', (muestra_id,))
```

## Notas

- **Offline:** el formulario lleva las clases `form-memoria` y `selector-memoria`, así que se guarda en la "mochila" como los demás. Las determinaciones viajan en un único campo oculto (`determinaciones`), que es justo lo que el sistema offline serializa bien.
- **Un informe = varias muestras:** este informe de INTA trae 16 muestras en un PDF. Con la carga manual, cargás cada una eligiendo su muestra y poniendo el mismo laboratorio/fechas y su Nº de protocolo. (El día que quieras automatizar esto con la API de Claude, el esquema ya está listo para recibirlo sin cambios.)
- **Editar:** por ahora guardar reemplaza el análisis de esa muestra (DELETE+INSERT). Si más adelante querés precargar el formulario al elegir una muestra ya cargada, se hace con un pequeño endpoint tipo `/api/datos_crudos` que devuelva las determinaciones.
